# Story 2.5: Login com Supabase Auth

## Story Statement

> **As a** operador da agência,
> **I want** autenticar por email e senha usando Supabase Auth,
> **So that** a sessão já use o provedor real previsto para produção.

## Acceptance Criteria

1. [x] O login usa Supabase Auth por email e senha.
2. [x] A sessão persiste token em cookie HTTP-only.
3. [x] A home valida a sessão consultando o usuário no Supabase Auth.
4. [x] A tela de login foi adaptada para email/senha.
5. [x] O dashboard exibe o email da sessão e permite logout.
6. [x] Os testes foram ajustados ao novo `SessionUser`.
7. [x] Checklist e file list foram atualizados.
