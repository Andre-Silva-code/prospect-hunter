# Story 1.2: Supabase Setup & NextAuth Configuration

## Story Statement

> **As a** developer,
> **I want** to configure Supabase PostgreSQL database and NextAuth.js v4 with Google OAuth authentication,
> **so that** users can securely authenticate with Google and their profiles are persisted with role-based access control enforced through RLS policies.

---

## Overview

This story establishes the authentication and data persistence layer for the Prospect Hunter application. It configures Supabase as the PostgreSQL database backend, implements NextAuth.js v4 for OAuth-based authentication, and sets up Row-Level Security (RLS) policies for multi-tenant data isolation.

**Key Deliverables:**

- ✅ Supabase project created and connected to Next.js app
- ✅ NextAuth.js v4 configured with Google OAuth provider
- ✅ User authentication table with appropriate schema and indexes
- ✅ RLS policies enforcing user data isolation
- ✅ Authentication middleware protecting dashboard and API routes
- ✅ Login/logout flows fully functional end-to-end
- ✅ Session management with JWT encryption and secure cookies
- ✅ Unit tests for auth endpoints and middleware
- ✅ Environment variables properly configured and documented

#### Supabase Schema Reference

**users table:**

| Column     | Type         | Constraints                            | Notes                      |
| ---------- | ------------ | -------------------------------------- | -------------------------- |
| id         | UUID         | PRIMARY KEY, DEFAULT gen_random_uuid() | Unique identifier          |
| email      | VARCHAR(255) | UNIQUE, NOT NULL                       | OAuth email from Google    |
| name       | VARCHAR(255) | NULL                                   | User's display name        |
| avatar_url | VARCHAR(500) | NULL                                   | Google profile picture URL |
| role       | VARCHAR(50)  | DEFAULT 'user'                         | user, admin, etc.          |
| created_at | TIMESTAMP    | DEFAULT now()                          | Account creation time      |
| updated_at | TIMESTAMP    | DEFAULT now()                          | Last modified time         |
| last_login | TIMESTAMP    | NULL                                   | Last successful login      |

**Indexes:**

- `idx_users_email` on `email` column (for fast OAuth lookups)

**RLS Policies:**

```sql
-- Users read own record
CREATE POLICY "Users can view own record" ON public.users
  FOR SELECT USING (auth.uid() = id);

-- Users update own record
CREATE POLICY "Users can update own record" ON public.users
  FOR UPDATE USING (auth.uid() = id);

-- Admins delete (service role bypass for migrations)
CREATE POLICY "Admins can delete" ON public.users
  FOR DELETE USING (auth.uid() = id AND role = 'admin');
```

#### NextAuth Configuration Details

**File:** `lib/auth.ts`

```typescript
import { NextAuthConfig } from "next-auth";
import GoogleProvider from "next-auth/providers/google";

export const authConfig: NextAuthConfig = {
  providers: [
    GoogleProvider({
      clientId: process.env.GOOGLE_CLIENT_ID!,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
    }),
  ],
  pages: {
    signIn: "/login",
    signOut: "/",
  },
  callbacks: {
    async signIn({ user }) {
      // User record created/updated in Supabase via signIn callback
      return true;
    },
    async jwt({ token, user }) {
      if (user) {
        token.id = user.id;
        token.email = user.email;
      }
      return token;
    },
    async session({ session, token }) {
      session.user.id = token.id as string;
      return session;
    },
  },
  session: {
    strategy: "jwt",
    maxAge: 30 * 24 * 60 * 60, // 30 days
  },
  cookies: {
    sessionToken: {
      name: "next-auth.session-token",
      options: {
        httpOnly: true,
        secure: process.env.NODE_ENV === "production",
        sameSite: "lax",
      },
    },
  },
};
```
