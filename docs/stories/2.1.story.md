# Story 2.1: Persistência de Leads em API Interna

## Story Statement

> **As a** dono de agência de tráfego pago,
> **I want** que os leads sejam persistidos no servidor da aplicação,
> **So that** o CRM não dependa mais só de `localStorage` do navegador.

## Acceptance Criteria

1. [x] Existe repositório server-side para listar, criar e atualizar leads.
2. [x] Existem rotas `GET/POST` para leads e `PATCH` para lead individual.
3. [x] O workspace carrega os leads a partir da API interna.
4. [x] Cadastro e atualizações do funil sincronizam com a API.
5. [x] O arquivo de dados persistido não é versionado.
6. [x] Existe teste cobrindo CRUD básico do repositório.
7. [x] Checklist e file list foram atualizados.

## Tasks / Subtasks

- [x] Criar repositório de leads com persistência em arquivo.
- [x] Criar rotas da API para listagem, criação e atualização.
- [x] Adaptar o workspace para usar `fetch`.
- [x] Proteger o repositório do versionamento do arquivo de dados.
- [x] Adicionar teste do repositório.
- [x] Validar com `npm run lint`, `npm run typecheck` e `npm test`.
