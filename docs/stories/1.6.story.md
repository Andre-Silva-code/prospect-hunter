# Story 1.6: Mini CRM Local para Operação de Prospecção

## Story Statement

> **As a** dono de agência de tráfego pago,
> **I want** cadastrar leads e acompanhar cada um por etapa do funil,
> **So that** eu consiga operar a prospecção diária com histórico local e sem depender ainda de backend.

## Acceptance Criteria

1. [x] Existe um formulário para cadastrar novos leads diretamente na interface.
2. [x] Os leads aparecem organizados em colunas de funil.
3. [x] Cada lead pode avançar de etapa sem recarregar a página.
4. [x] Os dados persistem em `localStorage`.
5. [x] O código isola tipos e lógica de armazenamento para facilitar futura troca por API.
6. [x] Existe teste cobrindo criação e avanço de lead.
7. [x] Checklist e file list foram atualizados.

## Tasks / Subtasks

- [x] Criar tipos de lead e etapa.
- [x] Implementar utilitários de persistência local.
- [x] Criar workspace client-side com formulário e pipeline.
