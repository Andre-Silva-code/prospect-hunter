# Story 1.5: MVP de Sistema de Prospecção para Agência de Tráfego Pago

## Story Statement

> **As a** dono de agência de tráfego pago,
> **I want** visualizar um sistema claro de prospecção com ICP, score, cadência e rotina operacional,
> **So that** eu consiga organizar a aquisição de novos clientes com previsibilidade.

## Acceptance Criteria

1. [x] A home deixa de ser uma canary page e apresenta um sistema de prospecção utilizável.
2. [x] O conteúdo é direcionado para agência de tráfego pago, com ICPs e ofertas coerentes com esse contexto.
3. [x] Existe uma lista de oportunidades com score e prioridade para facilitar qualificação.
4. [x] Existe uma cadência de abordagem multicanal com mensagens exemplo.
5. [x] Existe uma rotina operacional diária para transformar prospecção em processo.
6. [x] Há cobertura de teste para garantir que o dashboard principal renderiza os blocos-chave.
7. [x] Checklist e file list da story foram atualizados antes da conclusão.

## Tasks / Subtasks

- [x] Substituir a landing inicial por um dashboard com foco em prospecção.
- [x] Criar fonte única de dados estáticos para ICP, canais, prospects e sequências.
- [x] Ajustar tipografia e estilos globais para uma UI mais intencional.
- [x] Adicionar teste de renderização da página principal.
- [x] Corrigir configuração PostCSS para compatibilidade com ESM e Vitest.
- [x] Validar com `npm run lint`, `npm run typecheck` e `npm test`.

## Dev Notes

- Escopo definido como MVP front-end operacional, sem depender ainda das integrações de Apify, Supabase e Gemini previstas nas stories 1.2-1.4.
- A estrutura criada prepara o terreno para trocar dados estáticos por fontes reais nas próximas stories.
