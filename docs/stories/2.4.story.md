# Story 2.4: Autenticação Local com Sessão em Cookie

## Story Statement

> **As a** operador da agência,
> **I want** entrar no sistema com uma sessão real,
> **So that** o CRM deixe de depender de identificação manual no cliente.

## Acceptance Criteria

1. [x] Existe sessão local baseada em cookie HTTP-only.
2. [x] Existe rota de login e rota de logout.
3. [x] Existe página `/login`.
4. [x] A home redireciona para `/login` quando não existe sessão.
5. [x] O dashboard usa o usuário da sessão.
6. [x] O workspace deixa de depender do seletor manual de operador.
7. [x] Os testes foram ajustados ao novo contrato.
8. [x] Checklist e file list foram atualizados.
