# Story 2.6: Sidebar responsiva e navegação mobile

## Story Statement

> **As a** operador da agência,
> **I want** acessar o Prospect Hunter no celular com navegação funcional,
> **So that** posso usar o CRM e checar leads enquanto estou fora do computador.

## Contexto

O layout atual usa `flex min-h-screen` com sidebar fixa. Em telas menores que `lg` (1024px) a sidebar some completamente — o conteúdo ocupa tela inteira mas não há como navegar entre abas. O app é inutilizável no mobile.

## Acceptance Criteria

1. [ ] Em telas menores que `lg`, a sidebar é substituída por uma bottom navigation bar com os 4 ícones de navegação.
2. [ ] A bottom nav marca o item ativo com a cor accent (`#a04b2c`).
3. [ ] O conteúdo principal não fica coberto pela bottom nav (padding-bottom adequado).
4. [ ] Em `lg+`, o layout existente (sidebar lateral) é mantido sem alterações visuais.
5. [ ] O botão de logout é acessível no mobile (pode ser via menu ou ícone na bottom nav).
6. [ ] A transição entre sidebar desktop e bottom nav mobile não gera flash ou layout shift.

## Technical Notes

- Modificar `components/sidebar.tsx` e `app/(app)/layout.tsx`
- Bottom nav: `fixed bottom-0 left-0 right-0` com `z-50`, `bg-[#1c1410]`, `border-t border-white/[0.08]`
- Ícones: os mesmos SVGs já definidos em `NAV_ITEMS`
- Logout no mobile: adicionar ícone de saída como 5º item ou usar `...` menu
- Adicionar `pb-16 lg:pb-0` no `<main>` para evitar overlap com a bottom nav
