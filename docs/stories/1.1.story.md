# Story 1.1: Project Setup, Git, CI/CD, Environment

> **As a** developer,
> **I want** to initialize the Next.js project with Git, GitHub, ESLint, Prettier, TypeScript, TailwindCSS, and automated testing,

#### GitHub Actions Workflow

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  lint-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: "18"
      - run: npm ci
      - run: npm run lint
      - run: npm test
```

## Dev Notes for Implementation

### Important Setup Reminders

- Use Node.js 18 LTS (not 20+, for broader compatibility)
- `create-next-app` creates `.gitignore` automatically — verify `.env.local` is included
- ESLint config is auto-created but may need tweaking for stricter rules
- Husky requires git to be initialized BEFORE running `husky install`
- `.github/workflows/` directory must exist before creating workflow file

### Potential Pitfalls

- ❌ Forgetting to run `git init` before `husky install` → Husky install fails
- ❌ Committing `.env.local` to git → Credentials exposed (should be in `.gitignore`)
- ❌ Using `npm install` instead of `npm ci` in CI → Non-deterministic dependencies
- ❌ Workflow runs on `push` to any branch → Causes unnecessary CI runs
- ❌ Not setting `NEXTAUTH_SECRET` → Server will fail to start later

### GitHub Actions Tips

- Use `actions/checkout@v3` and `actions/setup-node@v3` (latest stable versions)
- `npm ci` (clean install) is preferred over `npm install` in CI for reproducibility
- Matrix builds (e.g., testing on multiple Node versions) can be added later
